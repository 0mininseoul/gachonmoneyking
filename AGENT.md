# AGENT.md (AI Agent Development Guide)

이 문서는 **Gachon Money King** 프로젝트를 유지보수하고 기능을 추가하는 AI 코딩 어시스턴트(Agent)를 위한 핵심 개발 가이드입니다. 

---

## 🎯 서비스 핵심 기획 및 정책 (Core Policy)
1. **자산 정보의 익명성**:
   - 실제 통장 잔고가 노출되는 서비스 특성상, 사용자들은 본인의 프라이버시를 극도로 숨기고자 합니다. 
   - 따라서 SNS 계정이나 성명 연상 닉네임 사용을 지양하고, **대학교 에브리타임 스타일의 유머러스하고 익명성이 강한 닉네임**을 유지해야 합니다.
2. **더미 데이터 품질**:
   - 총 16개의 정예 유학생 더미 데이터를 사용합니다 (베트남 7, 중국 5, 일본 1, 몽골 2, 우즈벡 1).
   - **원어 표기**: 각 나라 유학생의 닉네임은 영어 알파벳이 아닌 **모국어 문자(디아크리틱 베트남어, 일본어 가나, 중국어 간체, 몽골 키릴 문자)**로 자연스럽게 작성되어야 합니다.
   - **잔고 현실성**: 한국 내 유학생의 특성을 반영하여 대부분의 잔고는 **100만 원 이하(수만 원 ~ 80만 원 선)**로 책정하며, 100만 원 초과 자산가는 극소수(1~2명)로 설정해 극적인 리얼리티를 유지합니다.

## 🎨 디자인 시스템 및 CSS 가이드 (Vanilla CSS & Linear style)
- **테마**: Deep Dark 테마 (`--color-canvas: #010102`, `--color-surface-1: #0f1011`)
- **포인트 컬러**: Linear Lavender-Blue (`--color-primary: #5e6ad2`)
- **스타일 제약**: TailwindCSS 등의 라이브러리를 사용하지 않고, 오직 `src/index.css`에 모든 유틸리티와 컴포넌트 스타일을 모아 관리합니다.
- **테이블 정렬 일관성**:
  - 리더보드 테이블의 헤더(Header)와 값(Row value)의 정렬 축은 반드시 일치해야 합니다.
  - 이를 위해 `text-align`에만 의존하지 않고, 모든 컬럼 클래스(`.col-rank`, `.col-user`, `.col-nationality`, `.col-balance`)에 `flexbox` 정렬 설정을 완벽하게 매핑했습니다.
  - 순위: Center, 닉네임: Left, 국적: Center, 잔고: Right 정렬을 유지하십시오.

## 🌐 다국어 지원 (i18n) 및 예외 처리
- **지원 언어 (7개)**: 한국어(ko), 영어(en), 베트남어(vi), 중국어(zh), 몽골어(mn), 우즈베크어(uz), 일본어(ja).
- **레이아웃 깨짐 대응 (mn, uz)**:
  - 몽골어(mn)와 우즈베크어(uz)는 문자열의 길이가 타 언어에 비해 현저히 깁니다. 
  - 모바일 해상도에서 텍스트가 잘리는 현상을 방지하기 위해 `h1` 타이틀은 `clamp()` 폰트 스케일링과 `line-clamp` 속성을 적용해 최대 2줄 래핑을 지원합니다.
  - 모바일 테이블의 국적 헤더는 긴 텍스트 대신 `🌐` 아이콘으로 대체 처리되어 있습니다.
- **국가별 탭 및 선택기 정렬**:
  - 우측 상단 언어 드롭다운 메뉴 및 리더보드 국가 필터 탭 정렬 순서는 다음과 같이 고정합니다:
    `전체` → `베트남(vi)` → `중국(zh)` → `몽골(mn)` → `우즈벡(uz)` → `일본(ja)`

## 💾 Supabase 마이그레이션 및 Seeding 규칙
- 스키마 변경 또는 데이터 추가 시 `supabase/migrations/` 폴더 내에 순차적으로 마이그레이션 SQL 파일을 생성합니다.
- `profiles` 테이블의 `is_dummy = true` 컬럼을 통해 실제 가입 유저와 더미 유저를 명확히 구분합니다.
- 리더보드 정렬 시 로그인 유저는 탭 필터 종류에 상관없이 **최상단 고정(Pinning) 처리**되며, 중복 렌더링을 방지하기 위해 하단 일반 순위 리스트에서 본인의 ID는 제외한 뒤 연산하는 로직을 따릅니다.
